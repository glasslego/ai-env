# 스펙 정합성 리뷰 프롬프트

토픽: **{{topic_name}}** ({{topic_id}})

## 지시사항

SPEC과 구현 코드를 비교하여 정합성 리뷰를 수행하라.

### 입력
- SPEC 파일: `{{spec_file}}`
- 코드 저장소: `{{target_repo}}`
- 변경사항: `git diff` 또는 최근 커밋

### 출력
- REVIEW 파일: `{{review_output}}`

### 리뷰 규칙

1. **AC 매핑**: SPEC의 각 Acceptance Criteria가 코드/테스트로 충족되는지 확인
   - 충족: 코드 위치 + 테스트 파일 명시
   - 미충족: 구체적 누락 사항 기술

2. **Must Fix**: 스펙 위반, 버그, 보안 이슈
3. **Nice to Have**: 개선 제안, 리팩토링, 성능

4. **운영 관점 체크**:
   - 에러 처리: 예외 경로 커버리지
   - 로깅: 디버깅에 충분한 로그
   - 엣지 케이스: 경계값, 빈 입력 등

5. **Follow-up Tasks**: 리뷰에서 발견된 추가 작업 목록

### 토큰 효율 지침
- SPEC의 AC 섹션 먼저 읽고, 해당 코드만 탐색
- 전체 코드 읽지 말고 AC 관련 파일만 Read
- 리뷰 결과는 테이블 형식으로 압축
