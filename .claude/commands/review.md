# Code Review
---
description: 코드를 리뷰하고 잠재적 문제점과 개선사항을 제안합니다
---

## 입력
- 리뷰할 파일 경로 또는 변경사항

## 수행 단계

1. **코드 분석**
   - 변경된 파일 읽기
   - 로직 흐름 파악
   - 의존성 확인

2. **체크리스트 검토**
   - [ ] Type hints 있는가?
   - [ ] Docstrings 있는가?
   - [ ] 에러 처리가 적절한가?
   - [ ] 테스트가 있는가?
   - [ ] 보안 취약점은 없는가?
   - [ ] 성능 이슈는 없는가?

3. **악플(건설적 비판) 제공**
   PDF 권장: "악플을 요청하면 효과가 좋다"
   - 놓친 edge case
   - 잠재적 버그
   - 개선 가능한 부분
   - 다른 관점의 접근법

4. **결과 보고**
   - 🔴 Critical: 반드시 수정 필요
   - 🟡 Warning: 수정 권장
   - 🟢 Suggestion: 선택적 개선
   - 💡 Note: 참고사항
