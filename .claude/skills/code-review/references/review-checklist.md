# Review Checklist

## 코드 품질

- [ ] 함수당 20줄 이하 (초과 시 분리 검토)
- [ ] 들여쓰기 3단 이하 (early return 활용)
- [ ] 매직 넘버 없음 (상수 추출)
- [ ] 중복 코드 없음 (DRY)
- [ ] 사용하지 않는 import/변수/함수 없음

## 보안

- [ ] 하드코딩된 시크릿 없음
- [ ] SQL injection 방지 (파라미터 바인딩)
- [ ] XSS 방지 (출력 이스케이프)
- [ ] 입력 검증 (외부 입력 경계에서)

## 테스트

- [ ] 새 코드에 대한 테스트 존재
- [ ] 경계값/에러 케이스 테스트
- [ ] 외부 의존성 mock 처리
- [ ] 테스트 네이밍 컨벤션 준수

## 아키텍처

- [ ] 레이어 책임 분리 준수 (API ↛ DB 직접 접근)
- [ ] 의존성 방향 정확 (상위 → 하위)
- [ ] 인터페이스/프로토콜 활용 (필요시)

## 운영

- [ ] 적절한 로깅 (loguru, 디버깅 충분)
- [ ] 에러 처리 (외부 API/DB 호출부)
- [ ] 타임아웃 설정 (외부 호출)
- [ ] Graceful degradation (외부 서비스 장애 시)
