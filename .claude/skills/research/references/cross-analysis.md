# 4-Way Cross-Analysis Framework

리서치 결과를 종합할 때 반드시 4가지 관점을 적용한다.

## 1. 공통 발견 (Consensus)

2개 이상 소스가 동의하거나 유사하게 언급하는 사항.
신뢰도가 가장 높으므로 spec의 **핵심 근거**로 활용.

> 예: "Track A/B/C 모두 ccxt 라이브러리를 비동기 거래에 추천"
> → spec에서 ccxt 채택의 핵심 근거

## 2. 상충 의견 (Divergence)

소스 간 **다르게 말하는 부분**을 명시적으로 기록.

> 예: "Gemini는 Freqtrade 추천, GPT는 Jesse 추천"
> → 테이블로 비교, 채택 결정과 근거 명시

| 주제 | Source A | Source B | 채택 | 근거 |
|------|----------|----------|------|------|
| ... | ... | ... | ... | ... |

## 3. 고유 인사이트 (Unique)

특정 소스에서만 나온 중요 정보:

- **Gemini 고유**: 최신 시장 데이터, 한국 규제, 거래소 특이사항
- **GPT 고유**: 학술 논문 근거, 수학적 모델, 리스크 공식
- **Claude(Track A) 고유**: 특정 라이브러리 API 상세, GitHub 이슈
- **Manual 고유**: 실무 경험, 내부 데이터, 도메인 전문지식

## 4. 신뢰도 평가 (Confidence)

| 근거 유형 | 신뢰도 |
|-----------|--------|
| 출처 URL/논문 인용 있는 주장 | 높음 |
| 2개 이상 소스 일치 | 높음 |
| 일반적 서술만 있는 주장 | 중간 |
| 단일 소스, 인용 없음 | 낮음 |
| 소스 간 상충 | 낮음 (추가 검증 필요) |
